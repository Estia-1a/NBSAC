Outils de GitHub

Les principales étapes pour travailler avec un dépôt GitHub sont :


  • Clone : dupliquer le dépôt de GitHub vers votre machine

  • Modify : modifier les fichiers copiés en local

  • Stage : déterminer les changements que vous voulez synchroniser

  • Commit : envoyer vos modifications

  • Sync : synchronisr vos changements avec le dépôts


Différents outils

  • Dépôt : Le dépôt (repository) représente plus ou moins la totalité de la base de données que représente un système de gestion de version. Le dépôt contient toutes les versions (tout l'historique) d'un projet.

  • Copie de travail : La copie de travail (working copy, directory) représente le système de fichiers actuelle du projet (l'ensembles des fichiers dans le dossier et les sous-dossiers du projet).

  • Instantané : L'instantané (Snapshot) est l'état ponctuel des fichiers d'un système de fichiers (projet, dossier, sélection de fichiers, etc.). Il représente son contenu, à un moment donnée.

  • Commit : Un commit (validation), représente un ensemble de changements réalisés sur le dépôt (repository) et qui constitue une nouvelle version du dépôt.

  • Branche : Un ensemble de validations (commits) ayant divergé de la branche d'origine.

  • Système de gestion de version : Un système de gestion de version est donc constitué d'au moins un dépôt (repository) et d'une copie de travail (directory). L'ajout de nouvelles versions dans le système se fait grâce à la création de validations (commits) qui sont constitué de diffs entre les instantanés d'un commit précédent et de la copie de travail.

  • Fusions et demandes de fusions Merge Requests : La fusion peut être faite entre deux branches du même projets.

  • Les tickets (issues) : Ils permettent premièrement de gérer et corriger les bugs dans un projet.
